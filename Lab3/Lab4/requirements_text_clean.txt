Introduction
In this lab, you will write and test code on the ESP32 that modulates an audio signal using pulse width
modulation (PWM), allowing the ESP32 to output directly to a speaker. Additionally, you will use the
100 individually addressable WS2812 RGB LEDs to display a visualization of the audio signal.
This lab requires low-level programming of both the LEDC and RMT peripherals on the ESP32. The
RMT peripheral generates a signal consisting of bits that are each represented as a time duration spent
at logic-1 followed by a time duration spent at logic-0. TheLEDC peripheral generates a PWM signal
that is preconfigured with a specific frequency.
WS2812 Addressing
The WS2812 is a single device that includes red, green, and blue LEDs and an integrated controller that
accepts a serial data stream for setting the brightness of the individual LED colors. Upon receiving the
first 24-bit RGB value on power-up or aafter a reset signal is received, the WS2812 configures its LEDs
with the corresponding 24-bit color. Aafterward, subsequently-received 24-bit values are forwarded on
to the next WS2812 in series. This allows a single master to individually control all the WS2812s that are
connected in a string.
The WS2812 protocol is asynchronous and consists only of a single “data in” signal called DIN and “data
out” signal called DOUT . On power-on, the LEDs are oﬀ. At this point the WS2812 is in an “accept” state,
in which it monitors its DIN port for a 24-bit value with which to set the three LED intensities (red, green,
and blue). Each bit is conveyed to the WS2812 as a high (level-1) period followed by a low (level-0)
period. These periods are set depending on if the transmitted bit is a 1 or a 0.
Sending a 1-bit requires a high period, orT1H, of 800 ns and a low period, orT1Lof 450 ns, while a
0-bit requires a high period, orT0H, of 400 ns and a low period, orT0L, of 850 ns. This is shown in the
figure below.

From the perspective of an individual LED, when in the “accept state” , the DOUT port is driven to a
constant logic-0. Aafter 24 bits are received, the WS2812 transitions into the “pass-through state” , where
incoming bits are ignored for the purpose of setting the LED intensities and are instead forwarded to
the DOUT port, allowing a downstream WS2812 to use the bits to set its LED intensities.
If no bits arrive on DIN for at least 50,000 ns, the WS2812 transitions back into “accept” state, allowing
its LED intensities to be set again. In other words, not driving DIN for 50 microseconds resets the
controller but does not turn oﬀ the LEDs, which will remain at their previous setting.
This means that, for playing a video using these LEDs, a controller must budget enough time to configure
all the LEDs in the frame and have enough time left over to include the 50 us idle period to reset for the
next frame. Thus, for a desired reconfiguration rate of, for example 30 times per second (30 fps), there
can be no more than (1/30 - 50us) / (850 ns + 400 ns) = 26627 LEDs, forming, for example, a 30 fps 16:9
display having resolution:
1sqrt(26627 * 16/9) x sqrt(26627 * 9/16) = 217 x 122 pixels
Note that this limitation could be overcome by using multiple ESP32 controlling a single display, or
potentially from a single ESP32 using more than one RMT channel, though there may be other limiting
factors (see below).

The RMT has eight channels. Each channel can be clocked by the 80MHz APB_CLK, a 40MHz XTAL_CLK,
or a 17.5MHz RC_FAST_CLK. Whichever reference clock is selected is subjected to a clock divider.
The divided clock increments a counter whose output is used to determine whether the channel
output is driven to logic-1 or logic-0. The number of clock cycles corresponding to each logic level are
programmed into a memory called the RMT RAM.

Steps for Transmitting on RMT
The RMT RAM contains enough entries for 384 pulses, although we recommend using the sosoftware to
update the RMT RAM aafter every 24 bits are transmitted (e.g. aafter configuring each LED). This way, the
sosoftware will write 24 entries plus a zero termination into the RMT RAM, trigger the RMT to transmit,
perform other tasks during the 30 us required for the transmission, and then repeat to setup the next
LED. Aafter all LEDs are set up, the sosoftware will pause for at least 50 us to cause all LEDs to re-enter the
“accept” state.
To perform a transmission:
(1) write 24 entries in the RMT RAM,
(2) write zero termination in the 25th entry in the RMT RAM,
(3) set CONF_UPDATE (bit 24) to 1 in the RMT_CH0_CONF0_REG register,
(4) set RMT_TX_START_CH0 (bit 0) to 1 in the RMT_CH0_CONF0_REG register.

LEDC
The ESP32 LEDC (LED PWM Control) peripheral generates PWM (pulse-width modulation) signals that
can be used for a variety of purposes. As the name implies, it’s intended to be used for driving an LED
to vary its brightness, but it can be used for anything that needs a PWM signal, including a speaker,
motor, or power MOSFET .
As with the RMT , the LEDC can use the 80MHz APB_CLK, a 40MHz XTAL_CLK, or a 17.5MHz RC_FAST_CLK.
You can use 80 MHz clock for this lab. This clock is divided by the PWM clock divider, which is a (18, 8)
fixed-point value.
Then the PWM period is the time required to flip a counter having N bits, where N is defined by the
duty resolution. For example, the PWM period for the 80 MHz clock with a divider of 1 and 8-bit duty
resolution would be (8e7/1)/256 = 312.5 KHz.

Audio Visualization
There are 100 LEDs on the board, connected in row-major order (i.e. the upper-left corner is LED 0, the
lower-right corner is LED 99). For this lab, we will visualize the audio on the LEDs by lighting up the
LEDs in order from 0 to 99 according to the audio magnitude and also setting their color to indicate the
magnitude.
Audio visualization is generally performed using a log-scale plot, i.e. the magnitude is shown as
log(duty cycle/256), which for duty cycles 1 to 255 gives a range of -5.5 to 0. Each of the 100 LEDs can
represent a range of 1.1 starting at -5.5.
